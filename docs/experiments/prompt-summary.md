# Prompt Engineering Summary

Prompt engineering is the practice of designing, refining, and structuring the inputs given to a large language model in order to reliably elicit accurate, useful, and well-formatted outputs. It treats the prompt as a programmable interface, combining clear instructions, relevant context, examples, output-format specifications, and constraints to steer the model's behavior without changing its underlying weights. Effective prompts often layer techniques such as role assignment, step-by-step reasoning ("chain-of-thought"), few-shot examples, and explicit failure-handling rules, and are iterated against measurable evaluation cases. For example, instead of asking "Summarize this article," a prompt-engineered version might say: "You are a financial analyst. Read the article below and produce a 3-bullet summary; each bullet must be under 20 words and cite a number from the text. If no numbers are present, respond with 'INSUFFICIENT DATA'." — this version constrains role, format, length, grounding, and fallback behavior, dramatically improving output consistency.
