# Context Engineering

Context engineering is the practice of designing, assembling, and managing the information an LLM sees at inference time so it can perform a task reliably. As Andrej Karpathy framed it in June 2025, it is "the delicate art and science of filling the context window with just the right information for the next step" — encompassing instructions, retrieved documents, tool outputs, prior turns, and memory, all curated under tight token budgets. Where prompt engineering focuses on the wording of a single instruction, context engineering treats the entire window as a dynamic system: deciding what to include, what to compress, what to evict, and in what order. A concrete example is a coding agent that, before answering "fix the failing test," runs retrieval over the repo, pulls in the failing test file, the function under test, related type definitions, and the last error trace, while summarizing older chat turns — so the model receives a compact, task-relevant slice rather than the raw repository or full history.
